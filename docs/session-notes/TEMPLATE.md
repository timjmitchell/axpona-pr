## Usage Notes

**File naming:** `ISSUE-NN-short-description.md`

**When to create:**
- Working on a GitHub Issue (auto-create on first log entry)
- Ad-hoc exploratory session (via `/session` command or explicit request)

**When to append:**
- New day working on same issue → add new date section
- Issue closed → update status to Complete

**Ad-hoc sessions:** Use `YYYY-MM-DD-description.md` format when not tied to an issue.
